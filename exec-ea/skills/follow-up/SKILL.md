# Follow-up

Input: **"$ARGUMENTS"**

- If it describes a **new commitment**, log it in `commitments.md` (who / what / due / status OPEN), per `_protocol.md`.
- If it's **empty or asks for status**, review all rows in `commitments.md` and surface: **overdue → due today → due this week**. Flag what's slipping and offer to act (e.g. draft the reply).
- When a commitment is closed, mark it done and update `last_verified`.
